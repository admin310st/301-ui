TDS Status in 301.st

Public status page for the TDS area while the product continues to mature.

Current Position

TDS is the one major area that is not yet in the same documentation-ready state as the rest of the product. It exists in the product and is actively developing, but it should not be described like the fully settled redirects workflow.

What Is Safe to Say Publicly

  • TDS is an active product area inside 301.st
  • It is being shaped into the same operational model as the rest of the platform
  • The public docs should describe status and boundaries, not a final authoritative playbook

What We Avoid for Now

  • Deep SEO pages that imply final feature completeness
  • Workflow promises that may still move during rollout
  • Copy that competes with the stable redirects documentation

Recommended Public Framing

Keep TDS discoverable, but frame it as a status page and future-facing product area. Let redirects carry the main stable routing intent in public search traffic until TDS is fully ready.

FAQ

Is TDS available inside the product?

Yes, but this page deliberately avoids presenting it as the most mature workflow.

Should public search traffic land on TDS pages first?

No. Redirects and setup guides are better SEO entry points right now.

Will this page change later?

Yes. Once TDS is fully settled, this status page can grow into a fuller guide set.
