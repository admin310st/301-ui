Privacy Policy

How we handle your data — transparency first.

Last updated: December 12, 2025

Core Principle: You Own Your Data

301.st operates as an orchestration layer for your infrastructure. We don't own your domains, zones, or traffic. Our job is to help you manage integrations, not to become a data silo.

This policy explains what minimal data we must collect to provide the service, how we protect it, and your rights to access, modify, or delete it at any time.

1. Data We Collect

Account Information

  • Email address: Used for authentication, account recovery, and service notifications.
  • Password: Hashed using bcrypt/Argon2. We never store plaintext passwords.
  • Session tokens: Refresh tokens stored server-side (encrypted), access tokens client-side (short-lived).

Integration Credentials (Encrypted)

  • Cloudflare API Tokens: Scoped tokens (read zones, edit DNS, etc.). Encrypted at rest using AES-256-GCM.
  • Domain Registrar API Keys: For syncing domain status (Namecheap, NameSilo, etc.). Also encrypted at rest.
  • Third-party OAuth Tokens: If you connect GitHub, Google, or similar (future feature).

Configuration Data

  • Projects, Sites, Streams: Names, descriptions, associations between domains and logical groups.
  • Redirect Rules & TDS Logic: Your traffic distribution configurations.
  • Domain Metadata: Domain names, sync timestamps, status flags (active/paused/error).

Usage & Diagnostic Data (Minimal)

  • Login timestamps: For security monitoring (unusual login patterns).
  • API request logs: Stored for 30 days for debugging. Includes endpoint, timestamp, HTTP status. No request bodies.
  • Error reports: Stack traces (sanitized to remove sensitive values) for bug fixes.

2. Data We Don't Collect

  • Traffic Analytics: Visitor IPs, user agents, analytics stay in your Cloudflare account. We don't proxy or log end-user traffic.
  • DNS Zone Contents: We read DNS records via API when you trigger a sync, but don't permanently cache them.
  • Cloudflare Passwords: You authenticate to Cloudflare separately. We only store API tokens with limited scope.
  • Payment Info: If/when we add billing, we'll use Stripe. Credit card data never touches our servers.
  • Behavioral Tracking: No third-party analytics (Google Analytics, Facebook Pixel, etc.). No cross-site tracking cookies.

3. How We Protect Your Data

  • Encryption: API tokens encrypted at rest (AES-256-GCM). HTTPS/TLS 1.3 for all traffic.
  • Access Control: Database credentials use principle of least privilege. Application servers can't access encryption keys directly.
  • Token Scoping: We request minimum required Cloudflare permissions (e.g., "Zone:Read, DNS:Edit" for specific zones, not "Account:Edit").
  • Audit Logs: Critical operations (token creation, domain sync, user permission changes) logged for security review.
  • Regular Security Audits: Penetration testing, dependency scanning, code reviews. See Security page for details.

4. Data Retention & Deletion

Active accounts: We retain your configuration data for as long as your account is active.

Account deletion: When you delete your account, we immediately:

  • Revoke all API tokens (if possible via API; otherwise, you must manually revoke in Cloudflare/registrar dashboards)
  • Delete all projects, sites, streams, and redirect rules
  • Anonymize or delete your email and authentication credentials within 30 days
  • Purge API request logs older than 30 days

Legal retention: We may retain anonymized logs for up to 90 days if required by law enforcement or fraud investigation.

5. Your Rights (GDPR, CCPA Compliance)

  • Access: Request a copy of all data we store about you (JSON export available in account settings).
  • Rectification: Correct inaccurate data via account settings or by contacting support.
  • Erasure ("Right to be Forgotten"): Delete your account and all associated data at any time.
  • Portability: Export your project/site/domain configurations in JSON format.
  • Objection: Opt out of non-essential data processing (currently, we don't have any).

To exercise these rights, email privacy@301.st or use the account settings page.

6. Third-Party Services

We use the following third-party services (each has its own privacy policy):

  • Cloudflare Workers: Hosts our frontend and API. Cloudflare may log request metadata (IPs, timestamps) per their privacy policy.
  • Cloudflare Turnstile: CAPTCHA alternative for bot protection. Does not track users across sites.
  • PostgreSQL (hosted): Database provider (e.g., Neon, Supabase). Data encrypted at rest.

We do not share your API tokens, project configurations, or account details with third parties for marketing or advertising purposes.

7. Changes to This Policy

We may update this policy to reflect new features or legal requirements. When we make material changes, we'll:

  • Update the "Last updated" date at the top
  • Send an email notification if the change affects data handling practices
  • Maintain a changelog of policy updates (available on request)

Continued use of 301.st after policy updates constitutes acceptance. If you disagree, you can delete your account before changes take effect.

8. Contact & Questions

If you have questions about this privacy policy or how we handle your data:

  • Email: privacy@301.st
  • Security issues: See Security page for responsible disclosure
  • General support: See Documentation
