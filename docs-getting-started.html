Getting Started With 301.st

The shortest stable path from account creation to working redirect operations.

Quick Setup Path

  1. Create an account. Start with email or OAuth and complete the initial login flow.
  2. Connect Cloudflare. Add your Cloudflare account so 301.st can read zones and support routing workflows.
  3. Sync domains. Import the domains you want to manage from integrations into the domains table.
  4. Create a project. Group related assets before attaching sites or redirect work.
  5. Add a site if needed. Sites help organize destinations and site-linked routing flows.
  6. Launch redirects. Use the redirects page for the stable production workflow that is already shipped.

Who This Setup Fits

  • Operators managing multiple Cloudflare-backed domains
  • Affiliate and acquisition stacks that need organized redirect operations
  • Teams that want projects, domains, sites, and redirect workflows in one control plane

What Is Stable Today

  • Auth and account access
  • Cloudflare and registrar integrations
  • Domains table, inspector, filters, and bulk workflows
  • Projects and sites management
  • Redirect setup and Cloudflare sync

Recommended Order of Work

If you are starting from zero, do not begin with advanced routing ideas. First make integrations and domains trustworthy, then organize projects, then ship redirect workflows.

  • First: integrations and domain sync
  • Second: project and site structure
  • Third: redirect rollout and operational cleanup

FAQ

  • Do I need Cloudflare first?
    For the main workflow, yes. 301.st is built around Cloudflare-managed infrastructure.
  • Can I start with redirects only?
    Yes. Redirects are the best first shipped workflow after integrations and domain sync.
  • Should I start with TDS?
    No. Treat TDS as a separate in-progress area until rollout is fully complete.
